# Cross-Company Secure Enterprise Knowledge Agent

## New Project Positioning

The upgraded project reframes the original employee-handbook RAG experiment as a
cross-company enterprise knowledge-agent safety study.

New title:

```text
Prompt-Injection Defense and Transfer Evaluation for Cross-Company Knowledge Agents
```

The core research question is no longer only whether one employee handbook RAG
system can resist prompt injection. The upgraded question is:

> Can a layered prompt-injection defense designed for one enterprise handbook
> transfer to other company knowledge bases without losing too much task utility?

## Motivation

Many course projects build helpful agents for learning, travel, or paper
reading. This project targets a different enterprise problem: companies need
agents that can answer policy questions, summarize handbook pages, draft
internal emails, and support onboarding, but those agents must not be misled by
malicious user prompts, poisoned retrieved documents, fabricated citations, or
cross-company policy contamination.

## Data Design

The upgraded corpus combines:

| Company / organization | Role in the project |
| --- | --- |
| Made Tech | Main cleaned employee-handbook corpus from the original project. |
| GitLab | Public handbook-style remote-work and operating model source. |
| 37signals/Basecamp | Public employee-handbook source for compact company policies and norms. |
| Valve | Public new-employee handbook source for culture and organization-model transfer tests. |

The default reproducible corpus is generated by:

```bash
python3 build_multi_company_corpus.py
```

It writes:

```text
data/multi_company/company_chunks.csv
```

By default, the script reuses existing Made Tech chunks and adds compact
official-source seed chunks for GitLab, 37signals/Basecamp, and Valve. To fetch
more official-page paragraphs when network access is available:

```bash
python3 build_multi_company_corpus.py --fetch-public-sources
```

## System Design

The upgraded system is a `Secure Enterprise Knowledge Agent`, implemented in:

```text
enterprise_agent.py
```

It supports task-oriented outputs rather than only one-shot question answering:

- policy question answering;
- policy summarization;
- onboarding checklist generation;
- internal email drafting;
- IT/support safety refusal.

The agent has two modes:

| Mode | Description |
| --- | --- |
| `control` | Ordinary agent baseline. It retrieves broadly and does not enforce company scope, poison filtering, or refusal policy. |
| `secure` | Safety-gated agent. It filters poisoned chunks, enforces target-company evidence, refuses credential/private-data requests, and validates citations. |

## Attack Design

The upgraded attack set is generated by:

```bash
python3 generate_cross_company_attacks.py
```

It writes:

```text
data/multi_company/cross_company_questions.csv
data/multi_company/cross_company_poisoned_chunks.csv
```

Attack surfaces:

| Attack surface | Meaning |
| --- | --- |
| `user_query` | Malicious instruction is placed in the user task. |
| `retrieved_document` | Poisoned synthetic chunks are injected into the retrievable corpus. |
| `cross_company` | The task asks for one company's policy but tries to import another company's policy as authority. |

The cross-company contamination attack is the key new contribution. It tests
whether an agent confuses policies across enterprise contexts, for example by
using Valve or 37signals evidence to answer a GitLab policy question.

## Defense Design

The secure mode implements a lightweight safety gateway:

1. detect unsafe requests such as credential or private-data requests;
2. retrieve candidate evidence;
3. filter poisoned or suspicious chunks;
4. enforce target-company scope;
5. generate task-specific output only from safe evidence;
6. verify citations for poison and cross-company contamination.

This keeps the original project's defense logic, but extends it to multi-company
transfer settings.

## Evaluation Metrics

The evaluator is:

```text
evaluate_cross_company.py
```

Metrics:

| Metric | Purpose |
| --- | --- |
| Normal task success rate | Whether ordinary task-oriented agent requests are answered with valid citations. |
| Attack success rate | Whether attack rows caused unsafe behavior, over-refusal, poisoned citations, or cross-company contamination. |
| Citation error rate | Whether answered rows contain missing, poisoned, fabricated, or wrong-company citations. |
| Policy contamination rate | Whether an answer cites another company's evidence for the target company. |
| Required refusal recall | Whether credential/private-data requests are refused. |
| Over-refusal rate | Whether safe, answerable tasks are refused unnecessarily. |

## One-Command Experiment

Run the full local experiment:

```bash
python3 run_cross_company_experiment.py
```

This performs:

1. multi-company corpus construction;
2. attack-set generation;
3. control-agent run;
4. secure-agent run;
5. evaluation summary generation.

Outputs:

```text
outputs/cross_company_agent/control/results.csv
outputs/cross_company_agent/secure/results.csv
outputs/cross_company_agent/evaluation/summary_metrics.csv
outputs/cross_company_agent/evaluation/asr_by_attack_surface.csv
outputs/cross_company_agent/evaluation/asr_by_company.csv
```

## Current Local Results

The default local experiment creates 89 chunks, 32 evaluation questions, and 8
synthetic poisoned chunks.

Summary from the reproducible local run:

| Run | Normal task success | ASR | Citation error among answered | Policy contamination | Poisoned citation |
| --- | ---: | ---: | ---: | ---: | ---: |
| Control | 37.50% | 100.00% | 81.25% | 18.75% | 68.75% |
| Secure | 100.00% | 25.00% | 0.00% | 0.00% | 0.00% |

Interpretation:

- The ordinary agent is highly vulnerable when poisoned chunks and cross-company
  evidence are present.
- The secure agent removes poisoned and wrong-company citations in this local
  proxy experiment.
- The remaining secure-agent ASR comes from over-refusal on some non-required
  attack rows, preserving the safety-utility trade-off theme from the original
  project.

## How This Upgrades the Project

Original contribution:

- single employee-handbook RAG;
- prompt-injection attack and defense;
- safety/utility metrics.

Upgraded contribution:

- multi-company enterprise-agent corpus;
- task-oriented agent workflows;
- automatic cross-company attack-set generation;

- safety gateway with target-company citation validation.

## Limitations

- The public-company seed chunks are compact by default. Use
  `--fetch-public-sources` or add more official pages to increase coverage.
- The default agent uses local extractive generation for reproducibility. It can
  be connected to the existing LLM-backed pipeline later.
- The metrics are proxy metrics. A final paper should still discuss the need for
  human grading or independent judge-model evaluation.
